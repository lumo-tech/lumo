lumo.contrib.nn.functional.log_softmax_stable(input: Tensor, dim: int | None = -1)¶

Applies a softmax followed by a logarithm. Both CPU and GPU implementations of log_softmax use following formula: $log_softmax(a)=(a-C)−log(sum{exp(a-C)}), C = max(a)$ But, in early version of MPS backend, the implementation is log(softmax(x)), which may cause unstable result (inf). See https://github.com/pytorch/pytorch/issues/94043 for details.

For that reasom, I rewrite this function to temporarily avoid this problem. Before the MPS backend fix this implementation, you can use this function to workaround.

Parameters:

  • input – same as F.log_softmax

  • dim – same as F.log_softmax
